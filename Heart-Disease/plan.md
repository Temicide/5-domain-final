# Heart Disease Colab Notebook Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Google Colab-ready heart disease classifier that authenticates to Kaggle inside Colab, downloads and extracts the competition CSVs, validates data and predictions, trains strong tabular models with out-of-fold threshold tuning, and writes `/content/submission.csv` with complete `Yes`/`No` labels.

**Architecture:** Keep reusable implementation in one importable Python module with focused pure functions for Kaggle credential resolution, Colab/local path resolution, download/extract, validation, feature engineering, modeling, metrics, thresholding, ensembling, and submission writing. Generate a Colab notebook from the same module so local tests and Colab execution use identical logic.

**Tech Stack:** Python 3, pandas, numpy, scikit-learn, LightGBM, optional CatBoost, optional XGBoost, Kaggle CLI/API, pytest, nbformat.
